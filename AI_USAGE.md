### AI Suggestion Accepted

Several AI suggestions were directly useful and adopted after verfication, including:

- Organizing the project into seperate Django apps which helps in the future expansion and easy understanding.
- Use of service layer for the box recommendation logic instead of placing all logic inside views.
- Structuring REST API endpoint using Django Rest Framework.
- Writing unit test models, API endpoint and the Recommendation logic.
- Improve the overall project doucmentations.

Before  accepting any suggestions, they will be tested locally to ensure they work properly with in project.


### Modified AI Suggestions

Some suggestions require changes before they were used.

Exmaple included:

- Serializer fields were adjusted to match the final database model.
- Some API response were modified according to assessment requirements.
- Generated code was modified according to project coding style.
- A few test cases were updated after changes in recommendation logic.
- Documentation were rewritten to reflect the actual implementation istead of any genric documetation.

These are some modification which ensure the final project remained consistent and easy to understand.

### Rejected AI Suggestions

Not every suggestion was used some are accepted , some modified and some rejected.

Some suggestions were rejected because:

- They added some unnecessary complexity for scope of the project.
- They introduced some functionality that was not part of the project requirements.
- Some which do  not match with the required API behaviour.
- They conflicted with decision which are already made during development.

Only suggestions that improved the project without affecting  correctness or simplicity were include. 

### Developer Responsability 

All source code was:

- Review before being added to the project.
- Integrated manually into codebase.
- Executed and verified locally.
- Tested using Django's testing framework.
- Validated through manual API testing with Postman.

Developer (Ravinder Kumar) remained responsible for the all implementation decisions and the correctness of the submitted project.
